# BEHAVIOR EVALUATION CODE
## Law of Engine-to-Interface Mapping

### PREAMBLE
This Code defines exactly how the User Interface must react to the State Engine.
The UI is not permitted to interpret state creatively. It must execute the behaviors defined herein.

---

### ARTICLE I — CANONICAL STANDING STATES
The UI must recognize exactly five Standing Bands. All engine outputs map to one of these.

| Band | Visual Character | Motion Law |
| :--- | :--- | :--- |
| **1. STABLE** | Cool accent, controlled contrast | Slow, ambient drift |
| **2. ASCENDING** | Brighter accent, upward gradients | Gentle expansion |
| **3. RISK** | Heated edges, subtle instability | Low-frequency pulse |
| **4. BREACH** | High contrast, fractured framing | Sharp snap-in, interruption |
| **5. RECOVERY** | Muted saturation, reconstructive | Slow fade reconstruction |

### ARTICLE II — EVALUATION TRIGGERS
The following Engine Events must trigger immediate UI mutation:

*   **`ON_DAILY_EVALUATION`** → Full Authority Sweep + Verdict Panel.
*   **`STANDING_DELTA`** → Chromatic retint + Numeric Authority update.
*   **`RISK_THRESHOLD`** → Decay indicators activate.
*   **`BREACH_DECLARED`** → Instant Authority takeover + Motion interruption.

### ARTICLE III — COMPONENT OBLIGATIONS

#### III.1 The Obligation Stack
*   **Active**: Sorted by Priority.
*   **Breach**: Disabled. No fulfillment allowed.
*   **Recovery**: Gated. Condition-based only.

#### III.2 The Continuity Band
*   **Extension**: Physical segment addition.
*   **Fracture**: Visible break in geometry.
*   **Reconstruction**: New band start (old band persists as scar).

#### III.3 The Ledger
*   **Append**: Only on Daily Closure or Breach.
*   **Immutable**: Past entries must desaturate and resist interaction.

### ARTICLE IV — MOTION LAW
Motion is permitted ONLY in response to State Change.
*   **Improvement** = Expansion.
*   **Risk** = Tension/Vibration.
*   **Failure** = Rupture/Fracture.
*   **Time** = Aging/Desaturation.

### ARTICLE V — ENFORCEMENT
Any component failing to react to these signals as defined is non-compliant.
Visual consistency is subordinate to State Visibility.
