# KERNEL GOVERNANCE CODE
## Law of Component Behavior & Contributor Compliance

### PREAMBLE
This Code defines the laws that govern the IRON codebase itself. It establishes the supremacy of the Kernel over all modules and defines the non-negotiable standards for code contribution.

---

### ARTICLE I — KERNEL SUPREMACY
**I.1** — The Kernel is the only authority.
**I.2** — No module may bypass Kernel APIs to access state, storage, or time.
**I.3** — All Institutional State must be derived from the Kernel's Ledger.

### ARTICLE II — COMPONENT LAW
**II.1** — Every component must declare its jurisdiction (Access, Verdict, Visualization).
**II.2** — Components must not maintain private sovereign state. Local state is for UI transient only.
**II.3** — Components must subscribe to Kernel Events; they may not poll or query.

### ARTICLE III — CONTRIBUTOR CONSTRAINTS
**III.1** — No code may be merged without constitutional badge.
**III.2** — No feature may be added without an Enforcement clause.
**III.3** — "Utility" libraries are prohibited unless constitutionally scoped.

### ARTICLE IV — SOVEREIGNTY CHECKLIST
Before merge, every change must satisfy:
1.  **Does it enforce law?**
2.  **Does it preserve history?**
3.  **Is it derived from the Ledger?**
4.  **Does it respect Standing?**

If the answer to any is NO, the change is rejected.
